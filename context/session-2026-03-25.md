## v2.0.1 버그 수정 내역

### 1. `locky edit` httpx.ReadTimeout 수정
**파일**: `locky/agents/edit_agent.py`, `tools/ollama_client.py`

**원인**: `client.chat()`은 `stream: False` — Ollama가 응답 헤더를 300초 이내에 전송 안 함 (CPU 추론)

**해결**:
```python
# edit_agent.py — streaming으로 교체
response = "".join(
    client.stream(
        [{"role": "user", "content": prompt}],
        system=DIFF_SYSTEM,
        timeout=None,  # CPU 추론 시 무제한 대기
    )
)
```

```python
# ollama_client.py — timeout 파라미터 추가
def stream(self, messages: list, system: str = "", timeout: int | None = None):
    effective_timeout = timeout if timeout is not None else self.timeout
    with httpx.Client(timeout=effective_timeout) as client:
        ...
```

### 2. `locky commit` pathspec 버그 수정
**파일**: `actions/commit.py`

**원인**: `git status --porcelain` 파싱 시 `line[3:]`이 rename 항목에서 잘못된 경로 생성

**해결**: 파일별 `git add` 대신 `git add -u` (tracked) / `git add .` (untracked) 사용

### 3. `locky agent` ModuleNotFoundError 수정
**원인**: 구버전 editable install MAPPING에 `locky/` 패키지 미포함

**해결**: `.venv/bin/pip install -e .` 재실행으로 MAPPING 재생성

### 4. `tests/test_agents_edit.py` mock 불일치 수정
**원인**: `OllamaClient.chat` mock → `OllamaClient.stream`으로 교체 필요

**해결**:
```python
# 변경 전
patch("tools.ollama_client.OllamaClient.chat", return_value=diff_block),
# 변경 후
patch("tools.ollama_client.OllamaClient.stream", return_value=[diff_block]),
```

## 아키텍처 핵심 사항

### actions/ 패키지
- 각 모듈: `run(root: Path, **opts) -> dict` 시그니처
- 모듈 간 의존 없음 (단위 테스트 용이)
- `status` 키: `"ok"/"pass"/"clean"` → 녹색, `"nothing_to_commit"` → 노랑

### locky/ 패키지 (v2)
- `LockySession(workspace, model, history)` — 세션 상태
- `AskAgent(session).run(question)` — RAG 없이 파일 컨텍스트 직접 전달
- `EditAgent(session).run(instruction, file_path, dry_run)` — diff 생성·patch 적용

### Ollama 클라이언트
- `OllamaClient.chat()` — 동기, `stream: False` (짧은 응답용)
- `OllamaClient.stream()` — 동기 제너레이터, `stream: True` (긴 응답 / CPU 추론)
- `OllamaClient.stream_chat()` — 비동기 제너레이터 (asyncio 환경)
